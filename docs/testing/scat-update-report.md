# SCAT分析機能 修正レポート

大谷尚のSCAT（Steps for Coding and Theorization）に基づく質的分析フロー・出力形式・UI表示の修正を完了しました。

## 1. 概念・ラベルの再定義とUI反映
UI上の各ステップのラベルを正式なSCATの手続きに準拠させました。
- **Step 1**: ① 注目語句（データ中の着目語句）
- **Step 2**: ② 言い換え語句・データ外（Step1を言い換える語句）
- **Step 3**: ③ 説明語句・文脈（Step2を説明する背景・条件など）
- **Step 4**: ④ テーマ・構成概念（浮かび上がるテーマ）
- **Step 5**: ⑤ 疑問・課題メモ（疑問・未解決点）

各セグメントのアコーディオンUIにて、これらの中間生成物を一切省略せずに表示する（監査可能性の担保）設計に変更しています。

## 2. AI 自動分析機能（エンドポイントの実装）
`src/api/routes/openai.ts` に `POST /api/openai/scat-analysis` を追加しました。
- 入力された自由記述テキストをAIが自動的に**発話単位または意味のまとまり単位でセグメント化**（1セグメントに複数トピックが混ざる場合は再分割）します。
- 各セグメントに対して、上記のStep1〜4＋疑問メモを生成します。
- セグメント群の分析結果をもとに、全体を統合した**「ストーリーライン」**と、そこから抽象化された**「理論記述」**をそれぞれ独立して生成します。

## 3. 分析結果の保存と表示
- `src/api/routes/data.ts` を修正し、`scat_projects` テーブルに `storyline` と `theoretical_description` カラムを追加しました。
- UI側 (`SCATAnalysisPage.tsx`) に「理論・ストーリーライン」タブを新設し、生成されたストーリーラインと理論記述をテキスト形式で確認できるようにしました。
- 「AI一括分析」ボタンを設置し、長文を入力すると自動でセグメント化→コーディング→理論化まで一気通貫で行い、結果を保存・再描画するフローを実装しました。

## 4. CSVエクスポートの修正
- CSVダウンロード機能において、内部変数名ではなく要求されたカラム名 (`segment_id`, `raw_text`, `step1_focus_words`, `step2_outside_words`, `step3_explanatory_words`, `step4_theme_construct`, `step5_questions_issues`) がヘッダーに正しく出力されるように `downloadScatCSV` を修正しました。
