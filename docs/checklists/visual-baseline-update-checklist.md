# Visual Baseline Update Checklist

For Developers submitting a baseline update:

- [ ] Local tests run and failed as expected prior to snapshot update (confirming tests catch the change).
- [ ] Snapshots updated locally via `--update-snapshots`.
- [ ] Unrelated snapshots were NOT accidentally updated.
- [ ] PR created with the updated snapshot images committed.
- [ ] Visual diffs (Before/After) are attached or clearly explained in the PR description.
- [ ] Tech lead / UI owner reviewed and approved.
- [ ] Nightly CI passes after merge.
